# Route Selection UI

## Purpose

Reference for current route selection information hierarchy and interaction points.

## Current Layout Priorities

Main focus area:

- domain/route identity
- route progression graph
- current reachable choices

Compact run context:

- hero summary (HP, level, XP)
- shard progress and forge readiness
- active boon summary
- compact run summary panel (deck and relic counts)

## Interaction and Inspection

- Node choice buttons are primary interaction.
- Reward badges telegraph route node categories.
- Lightweight inspections are available for:
  - deck summary
  - relic summary
  - boon details

## Direction Alignment

This UI follows playtest direction to keep decision flow clear while still exposing key run state in compact, inspectable form.
